What was added to the server:

New RPC Event Handlers:

1.  handleRpcRequestFullState - Players request full sync from DM
2.  handleRpcBroadcastFullState - DM broadcasts complete game state to specific/all players
3.  handleRpcSetUiMode - DM controls UI mode for all players
4.  handleRpcOpenCard - DM opens specific cards on player screens
5.  handleRpcSyncInitiativeOrder - Syncs initiative order across all clients

New SessionManager Methods:

-   getSocketIdByUserId() - Find socket ID for a specific user
-   getDMsInSession() - Find all DMs in a session

Event Flow:

-   rpc_request_full_state → server notifies DM via full_state_requested
-   rpc_broadcast_full_state → server sends full_state_broadcasted to target users
-   rpc_set_ui_mode → server sends ui_mode_changed to players
-   rpc_open_card → server sends card_opened to players
-   rpc_sync_initiative_order → server sends initiative_order_synced to players

All RPC handlers include permission checks and support both targeted (specific user) and broadcast (all users) modes!
